# Requirements Spec

## Business Scenario

企业制度知识库和工单 AI 助手面向客服、运营和质检场景。系统要能根据工单问题检索企业制度、查询业务上下文、生成处理建议，并在高风险动作前要求人工确认。

核心场景：

- 客服打开退款工单，系统生成工单摘要。
- 系统检索退款制度、物流规则和历史处理口径。
- Agent 通过受控 Tool 查询订单、用户和工单状态。
- 高金额退款、关闭工单、转派工单等动作只生成建议，不直接执行写操作。
- 所有模型调用、RAG 检索、Tool 调用和 Agent 步骤进入 trace。

## User Scope

首批用户只覆盖内部客服和质检人员，不面向外部客户直接开放。

- 客服：查看工单处理建议、引用依据和风险提示。
- 质检：查看 AI 建议、Tool 调用链路和坏 case。
- 管理员：维护制度文档、查看评测报告和线上反馈。

## Risk Boundary

AI 只能生成建议，不能绕过业务系统权限执行写操作。

- 无引用依据时，RAG 不生成确定性制度结论。
- 检索到冲突制度时，返回人工确认提示。
- Tool API 默认只读；写操作必须有确认 token 和业务系统审批。
- Prompt、RAG、Tool、Agent 结果必须可追踪、可复盘。
