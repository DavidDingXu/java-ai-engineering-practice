# Eval Plan

## Golden Set

Golden Set 覆盖制度问答、工单建议和 Agent 工具调用路径。

样本至少包含：

- 正常退款制度问答。
- 发货后退款制度问答。
- 无依据问题。
- 冲突制度问题。
- 高金额退款工单。
- 工具参数错误。
- 需要人工确认的写操作。

每条样本记录输入、期望引用、期望风险等级、期望工具调用路径和人工复核结论。

## Regression

以下变更必须跑回归：

- Prompt 模板变化。
- 模型版本变化。
- RAG 切分策略变化。
- Rerank 策略变化。
- Tool schema 变化。
- Agent 规划逻辑变化。

回归结果至少比较通过率、引用准确率、工具路径准确率、平均 token 和平均延迟。

## Human Review

LLM-as-Judge 只能做初筛，不能替代人工抽检。

人工复核重点：

- 答案是否越权。
- 引用是否真实支持结论。
- 高风险动作是否要求人工确认。
- 工具调用路径是否符合业务规则。
- 坏 case 是否进入反馈和回归样本。
